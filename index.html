webrtc · zero-server

Messages that
touch no server.

Two browsers connect directly over a WebRTC data channel (SCTP over DTLS). The signaling server is used only for the handshake — once the channel opens, every message flows peer-to-peer, encrypted, never stored, gone when the tab closes.

properties
Privacy by architecture.
🔒

DTLS encrypted

All data is encrypted in transit; the signaling server never sees message content.

🚫

No persistence

Nothing is written to any backend. Refresh the tab and the conversation is gone.

🆔

No account

A random 3-character peer ID, shared out-of-band. No email, no login, no tracking.

📡

NAT traversal

STUN-based hole-punching via PeerJS; ordered SCTP delivery keeps messages in sequence.
